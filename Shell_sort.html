Shell Sort:

for(p = n/2; p > 0; p --)

{

for(i = p; i < n; i++)

{

temp=x[i];

for (j = i; j >= p && temp < x[j - p]; j -= p)

{

x[j] = x[j-p];

}

x[j] = temp;

}

}

PREZENTAREA METODEI

                Algorutmul shell sort este o generalizare a algoritmului insertion sort. La algoritmul insertion sort, pentru a insera in lista de elemente deja sortate, se deplaseaza fiecare element cu cate o pozitie spre dreapta atata timp cat avem elemente mai mari decat el.Practic fiecare element inainteaza spre pozitia lui finala cu cate o pozitie.
                Algoritmul shell sort lucreaza similar, doar ca deplaseaza elementele spre pozitia finala cu mai mult de o pozitie.Se lucreaza in iteratii. In prima iteratie se aplica se aplica un insertion sort cu un salt s1 mia mare decat 1. Asta inseamna ca fiecare element din sirul initial este deplasat spre stanga cu cate s1 pozitii atata timp cat intalneste elemente mai mari decat el.
                Se repeta asemenea iteratii cu salturi din ce in ce mai mici s2, s3, s4 etc. Ultima iteratie se face cu saltul 1. Aceasta ultima iteratie este practic un insertion sort clasic. Principiul este ca dupa fiecare iteratie sirul devine din ce in ce "mai sortat". Iar cum algoritmul insertion sort functioneaza cu atat mai repede cu cat sirul este mai sortat, per ansamblu vom obtine o imbunatatire de viteza.

PREZENTAREA ALGORITMULUI

                Algoritmul de sortarea prin metoda shell va sorta elementele din vectorul v1, v2 ,...,vn astfel incat dupa sortare, acestea vor fi în ordinea v1 ≤ v2 ≤... ≤ vn. Ne propunem sa facem o ordonare crescatoare a vectorului v1, v2 ,...,vn. Fiind o genralizare a algoritmului prin insertie, shell sort va lucra similar acestuia in iteratii. Cu fiecare iteratie se vor interschimba elemente din ce in ce mai apropiate astfel incat la iteratia finala cu pas 1 tabloul va fi suficient de sortat astfel incat timpul de executie al iteratiei finale sa se imbunatateasca, tinzand spre O(N);

Timpuri de executare:

  1. Cazul mediu : O( N*(log(N)))^2)
  2. Cazul defavorabil : O( N*(log(N))^2)
  3. Cazul cel mai bun: O( N*log(N))
  4. Memorie folosita : O(1)
  5. Stabil : NU
  6. Sortare descrescatoare : j>=p   &&   x[j-1]<v
  7. Sortare crescatoare : j>=p   &&   x[j-1]>v;
